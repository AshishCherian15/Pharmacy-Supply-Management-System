## 🧭 Overview

> **Pharmacy Supply Management System** is a full-stack Flask web application designed to manage daily pharmacy supply operations with role-based access control.

Started as a DBMS mini-project and upgraded into a **deployment-ready product** with:

- 🗄️ Production database on **Neon PostgreSQL**
- ☁️ Serverless deployment on **Vercel**
- 🔐 **Admin** and **Customer** portals with session security
- 📊 Chart-driven analytics and reporting
- 📥 CSV export, print-friendly order reports, and audit logs

## ✨ Feature Showcase

### 🛡️ Admin Workspace

<div align="center">

| 🧩 Module | 📋 What it does | 🔗 Route |
|:---|:---|:---:|
| 📊 Dashboard | KPI cards + status & monthly charts | `/dashboard` |
| 🏥 Pharmacy Registry | Add / edit / delete pharmacy stores | `/insert`, `/edit/{id}` |
| 📦 Order Management | Create orders, filter, status updates, delete | `/medicines`, `/orders` |
| 💊 Inventory Catalog | Manage medicines and products | `/items`, `/items2` |
| 🔍 Search | Fast search across catalog and records | `/search` |
| 📈 Reports | Revenue and operational insights | `/reports` |
| 📝 Logs | Auditable action history with export | `/logs` |

</div>

### 🛒 Customer Workspace

<div align="center">

| 🧩 Module | 📋 What it does | 🔗 Route |
|:---|:---|:---:|
| 📝 Register / Login | Customer self-onboarding and access | `/customer/register`, `/customer/login` |
| 🛍️ Catalog | Browse medicines with autocomplete | `/customer/catalog` |
| 📋 My Orders | Track status and cancel pending orders | `/customer/orders` |
| 📥 CSV Export | Export customer order history | `/customer/orders/export/csv` |

</div>

### 🔒 Security & Data Integrity

- 🔑 Password hashing via `werkzeug.security`
- 🛡️ CSRF protection on all POST forms
- 🚧 Role guards for admin / customer routes
- ⏱️ Session timeout for inactive users
- 🗄️ SQLAlchemy ORM — zero raw SQL

## ⚙️ Local Setup

### 1️⃣ Clone

```bash
git clone https://github.com/AshishCherian15/Pharmacy-Supply-Management-System.git
cd Pharmacy-Supply-Management-System
```

### 2️⃣ Create Virtual Environment

```bash
# 🪟 Windows
python -m venv venv
venv\Scripts\activate

# 🍎 macOS / Linux
python3 -m venv venv
source venv/bin/activate
```

### 3️⃣ Install Dependencies

```bash
pip install -r requirements.txt
```

### 4️⃣ Configure `.env`

```env
SECRET_KEY=your-random-secret-key
ADMIN_USERNAME=owner
ADMIN_PASSWORD=ChangeMe@123
SEED_DATA=true
# Leave DATABASE_URL blank to use local SQLite
```

### 5️⃣ Run 🚀

```bash
python run.py
```

Open 👉 **http://localhost:5000**

## 🚀 Deployment

### Vercel + Neon PostgreSQL

1. 🗄️ Create a **Neon DB** and copy the connection string
2. 📦 Import your repository in **Vercel**
3. ➕ Add these environment variables in Vercel dashboard:

<div align="center">

| 🔑 Key | 💡 Value |
|:---:|:---:|
| `SECRET_KEY` | strong-random-secret |
| `ADMIN_USERNAME` | `owner` |
| `ADMIN_PASSWORD` | your-secure-password |
| `DATABASE_URL` | Neon `postgres://...` URL |
| `SEED_DATA` | `true` |

</div>

4. ✅ Deploy and verify `/api/health`

> 💡 **Auto-deploy:** Every `git push` to `main` triggers an automatic Vercel redeploy — your live site updates instantly!
